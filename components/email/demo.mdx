## Integration with Forms

Here's how to integrate with a form handler:

```jsx
function SignupForm() {
  const [formData, setFormData] = useState({
    name: '',
    email: '',
    isValid: false
  });

  const handleValidation = (result) => {
    setFormData(prev => ({
      ...prev,
      isValid: result.isValid
    }));
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    if (formData.isValid) {
      // Submit form data
      console.log('Form submitted:', formData);
    }
  };

  return (
    <form onSubmit={handleSubmit} className="space-y-4">
      <div>
        <label className="block mb-1">Name</label>
        <input
          type="text"
          value={formData.name}
          onChange={(e) => setFormData({...formData, name: e.target.value})}
          className="w-full p-2 border rounded"
          required
        />
      </div>
      <div>
        <label className="block mb-1">Email</label>
        <EmailValidator 
          onValidate={handleValidation}
          initialValue={formData.email}
          onChange={(email) => setFormData({...formData, email})}
        />
      </div>
      <button 
        type="submit" 
        disabled={!formData.isValid}
        className={`px-4 py-2 rounded ${
          formData.isValid 
            ? 'bg-blue-600 hover:bg-blue-700 text-white' 
            : 'bg-gray-300 text-gray-500 cursor-not-allowed'
        }`}
      >
        Sign Up
      </button>
    </form>
  );
}
```

## Best Practices

1. **Always validate on the server** - Client-side validation is for UX, not security
2. **Provide clear error messages** - Help users understand and fix issues
3. **Use proper input types** - `type="email"` for better mobile keyboard support
4. **Consider accessibility** - Ensure proper labels and ARIA attributes
5. **Test thoroughly** - Check with different email formats and edge cases

## Troubleshooting

If the component isn't working as expected:

3. Verify that React is properly imported
4. Make sure you're not using conflicting CSS styles
